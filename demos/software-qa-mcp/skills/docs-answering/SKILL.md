# Docs Answering

Use this skill when answering software questions where documentation currency
matters.

## Workflow

1. Identify the product, library, framework, or API being asked about.
2. Use available MCP documentation tools before answering.
3. Prefer the smallest relevant doc-backed answer.
4. If there is version ambiguity, state the assumed version family.
5. Mark any inference as inference rather than presenting it as documented fact.

## Output Pattern

- `Answer:` concise answer first
- `Example:` only if it helps
- `Version note:` only when relevant
- `Source:` mention the MCP documentation source used

## When to escalate

Delegate to `docs-checker` when:

- the answer depends on exact option names or signatures
- the topic involves migration or deprecation risk
- two frameworks or versions are easy to confuse
